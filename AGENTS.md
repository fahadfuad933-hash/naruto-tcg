# NARUTO TGC — Chakra-Duell

Naruto-Kartenspiel im Yu-Gi-Oh-Duel-Links-Stil (Speed Duel: 8000 LP, 3 Monster- + 3 Jutsu/Fallen-Zonen, Tribute ab Stufe 5). Reine Web-App für Smartphone-Browser (Querformat optimiert), komplett offline, keine Dependencies, keine Build-Tools. UI-Texte auf Deutsch.

## Konventionen

- Vanilla JS, jedes Modul eine IIFE: `(function (g) { ... })(typeof window !== 'undefined' ? window : globalThis)` — Namespace `window.NTCG` / `globalThis.NTCG`
- `engine.js` und `ai.js` müssen DOM-frei bleiben (Node-Tests importieren sie direkt)
- Duell-Layout: 3D-Arena (`#board-scene` mit `perspective`, `#board-3d` mit `rotateX(var(--tilt))` — 38° Quer / 18° Hoch, `preserve-3d`-Kette bis `.slot`); Feldkarten: **Angriffs-Karten stehen aufrecht** (`.slot .card` counter-rotiert + `drop-shadow`), **Verteidigungs-Karten (defup + defdown) liegen quer und flach** auf der Matte (`.slot .card.def { rotate(90deg) scale(0.86) }` — KEINE Gegendrehung der Innerelemente, Bild/Text drehen sich mit; YGO-Look, Position auf einen Blick lesbar). HUD-Elemente absolut positioniert (große Touch-Ziele: LP-Balken 26px, Chips ~14.5px, Phasen-Button 60px). Hand als überlappender Fächer mittig unten (27 %/19 % der Höhe, Überlappung 0.30). `fitLayout()` in duel.js berechnet `--card-w`/`--hand-w`; `TILT`-Faktoren (0.82/0.90) + `reserve` (32/92) müssen zu CSS-`--tilt` + `perspective` passen — nach Änderungen mit `test/mkdebug.js` vermessen (Slot-Bottom der nahen Reihe darf Hand-Oberkante kaum überschreiten). **Klick-Handler hängen an der KARTE, nicht am Slot** (`card.onclick`): die aufrechte Karte ragt über die flache Slot-Box hinaus — Hit-Testing folgt der transformierten sichtbaren Form (Probe: `test/mkhit.js` → HIT|OK). Proben müssen daher `.slot .card` anklicken, nicht `.slot`
- Jutsu-/Fallenkarten haben kein `attr` — `attrVar()` (duel.js) liefert dann Türkis/Magenta. Pop-in-Animation nur über `.pop`-Klasse (`markNew`/`prevKeys`), nie pauschal auf `.card`
- Karten-Enthüllung (Flip durch Angriff, aktivierte Jutsu/Fallen) läuft über die `revealCard()`-Queue (duel.js) → Overlay `#reveal` mit Großkarte + Caption, reagiert auf Engine-Events `flip`/`activate`/`trap`. Verdeckt **setzen** zeigt nie die Karte
- Kampflog `#vlog` links (Vorschau der letzten 5 Aktionen, neueste oben, `vlogAdd()` in duel.js auf Engine-Events; Mini-Karten via `.vlog-thumb`-Wrapper, da `.card` den Eltern-Container füllt). Nur Querformat sichtbar; Spielerleiste im Querformat unten links verankert. **Tap auf `#vlog` öffnet das Log-Modal** (`showLogModal`, komplette Historie `D.vlogAll` max. 200 mit Zug-Nummer + Akteur, scrollen); Eintrag mit Karte antippen → `NT.CardView.show(id)` Großansicht (`#sheet` z-index 62 liegt über `#modal`)
- Karten-Design: Vollbild-Art über `.c-art` (Bild aus `NT.CARD_IMG`, sonst Emoji-Fallback), darüber Attr-Chip/Stufe/Name/Stats absolut; Raritäts-Rahmen mit Foil-Sweep bei SR/UR (`.card::after`, bei `.facedown` aus). `.card.def` dreht `.c-art img` mit `rotate(-90deg) scale(1.45)` gegen
- Kampf-FX (duel.js `fxSpawn`/`fxShot`/`fxPulse`): `.fx-slash` Angriff auf Ninja · `.fx-shot` Projektil bei Direktangriff + Karten-Zug (Deck-Chip → Hand) · `.fx-boom` Zerstörung · `.fx-ring` Beschwörung · `.fx-flash` Jutsu-/Fallen-Aktivierung · `.slot-hit`/`.lp-flash-d`/`.lp-flash-h`; LP-Zahlen rollen per rAF in `lpBar()`. **Zerstörungs-Animation:** `D.dying` (950 ms) — Geist-Karte `.card-dying` über dem leeren Slot (Shake → Sinken/Fade, 2 `.shard`-Splitter), wird in `slotEl` über leeren Monster-Zonen gerendert
- **Long-Press (480 ms) zeigt die Karte groß:** `attachLongPress(el, fn)` (duel.js, als `NT.attachLongPress` exportiert) — unterdrückt den Folge-Klick per Capture-Listener, daher VOR dem `onclick` registrieren. Verdrahtet: Hand, eigene + aufgedeckte gegnerische Feld-/ST-Karten, Deck-Editor (über `NT.CardView.show(id)` = Sheet ohne Aktionen)
- **Flow-Falle (gelöst):** Siege, die durch eine Fenster-Auflösung entstehen (KI passt im Fallen-Fenster → `resolveAttack`), laufen über `handleWindow → continueFlow` — dort muss `checkWin()` zuerst aufgerufen werden; zusätzlich Sicherheitsnetz `case 'win'` in `handleEvent` (verzögert `checkWin`)
- Bei Engine-Fehlern: deutsche `Error`-Message werfen, die UI zeigt sie als Toast

Für echte Bild-Screenshots eine `.gen.html` im **Projektroot** bauen (Pfade `js/` statt `../js/`). **Ausgabe immer per String-Konkatenation bauen** (`'…' + body + test + '…'`), NIEMALS `html.replace(...)`: im Ersatz-String kollabiert `$$` zu `$` und zerstört den `$$ = querySelectorAll`-Helper im Probe-Code (passiert bei mkshop).
